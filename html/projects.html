2D Game Engine

As part of a semester-long project, I am implementing a comprehensive 2D game engine from scratch using Java. My engine includes Systems to render game objects, handle user input, detect in-game collisions, and many more capabilities.

Essential to the implementation of this engine is an understanding of the flow of execution. In my design, program execution starts at the Application, then percolates through layers of increasing specificity and intricacy.

Take the journey of a single key press for example.

When the user presses the 'a' key, the current running Application passes the event along to the GameWorld, current active Screen, and ViewPort. The GameWorld passes execution along to its Systems, which pass executions to specific behaviors, which in turn update the state of the GameObjects to which they belong. The Screen and ViewPort pass execution along in a similar fashion.

I have built three games thus far on top of my engine:

Tic Tac Toe: a digital recreation of the classic game

Alchemy: my rendition of this game

Paws off my Crops: a work in progress involving tending to, and protecting, a farm.
